Présentation

Définition

Merise est une méthode d'analyse, de conception et de gestion de projet informatique.

Wikipedia

La méthode d'analyse Merise permet de faciliter et normaliser le processus de gestion de projet informatique et plus particulièrement la partie conception de base de données.

Historique

  • Créée dans les années 70
  • Méthode franco-française
  • Aujourd'hui souvent remplacée par la méthode UML
  • ... mais cependant toujours utile pour la conception de base de données

Le stockage des données

La majorité des outils informatiques ont besoin de stocker des données de manière permanente (non volatile).

Il est donc nécessaire d'effectuer des opérations d'écriture / lecture sur un disque dur.

Il est possible de stocker les données sous différents formats (CSV, XML, texte...).

On utilise généralement un service qui va se charger de stocker ces données (MySQL, SQL Server, Oracle...). On parle alors de Système de Gestion de Base de Données Relationnelle (SGBDR).

Etude conceptuelle

L'objectif est de décrire la structure et l'organisation des données de notre futur application.

Pour cela, nous allons nous appuyer sur deux diagrammes :

  • MCD (Modèle Conceptuel des Données) : Diagramme représentant les différentes entités reliées entre-elles par des associations.
  • MLD (Modèle Logique des Données) : Diagramme représentant les différentes tables qui devront ensuite être créées en base de données.
